# meeting-item-tracking



## 📊 第一阶段：多维表格字段设计

早会表格的核心是把“说了什么”转化成“谁去落实”。

| 字段名称 | 类型 | 说明 |
| --- | --- | --- |
| **会议日期** | 日期 | 默认创建日期 |
| **议题/项目** | 文本/单选 | AI 提取讨论的核心项目 |
| **待办事项 (Task)** | 长文本 | **核心字段**：AI 总结的具体执行动作 |
| **责任人** | 人员 | AI 识别姓名并匹配通讯录 |
| **完成时限** | 日期 | AI 识别（如“下周三前”、“明天”） |
| **执行状态** | 单选 | 未开始、进行中、已完成、延期 |
| **昨日遗留** | 复选框 | 用于区分是新任务还是旧任务（可自动关联） |

## 🤖 第二阶段：AI 自动录入逻辑

早会通常有几种形式，对应不同的抓取方式：

### 场景 A：群内文字/语音转文字总结

如果你在群里发一段早会文字纪要，或直接发送企业微信的语音消息：

* **AI Prompt 设置**：
> "你是一个秘书。请分析这段早会记录，将其拆解为多个任务。每个任务必须包含：1.项目名称；2.具体动作；3.负责人（必须是姓名）；4.截止时间。如果没提到截止时间，默认设为‘今日下班前’。请以表格结构输出。"


* **自动化流**：企业微信群消息 -> AI 提取 -> **多维表格“新增多行记录”**。

### 场景 B：语音通话/视频会议

利用企业微信会议的“自动会议纪要”功能：

1. 会议结束后，复制文本。
2. 粘贴到多维表格的“智能助手”中，让它一键生成任务记录。

## 🔄 第三阶段：关键功能实现

### 1. “昨日未完事项”自动同步（核心难点）

为了避免早会流于形式，需要追踪昨天没做完的事。

* **设置**：在自动化流中增加一个步骤。
* **逻辑**：每天早会前 10 分钟，系统自动筛选“状态 != 已完成”的任务，汇总成一条消息发到群里：“**今日早会重点追踪昨日遗留：[任务 A - 负责人]、[任务 B - 负责人]...**”

### 2. AI 自动分配任务

如果早会纪要里写的是“小张去跟进一下”，AI 需要精准识别“小张”并关联到通讯录里的“张三”。

* **技巧**：在多维表格中建立一个“花名册”工作表。AI 在处理时，先比对花名册，确保“负责人”字段能直接触发企业微信通知。

### 3. 即时任务派发

* **自动化触发**：当表格新增一条“待办事项”时。
* **动作**：自动给负责人发送一条企业微信卡片消息。
> **[任务指派通知]**
> 项目：XX设备调试
> 内容：检查传感器精度并反馈报告
> 时限：今日 18:00
> [点击确认领用]

## ⏰ 第四阶段：追踪与闭环

### 1. 晚间进度回填

每天 17:30，机器人私聊负责人：
“**[早会任务进度确认]**：你今天领取的‘XX任务’完成了吗？请回复：‘已完成’或‘延期+理由’。”

* **高级玩法**：负责人的回复可以直接通过机器人更新回多维表格的“执行状态”和“备注”列。

### 2. 领导日报看板

多维表格自带“仪表盘”功能：

* 创建一个图表，显示：**今日任务完成率**、**谁的延期最多**、**各项目进度分布**。
* 将仪表盘链接推送到部门领导的工作台。
